# 소개 영상 샷리스트 (지자체/공공 실무자 대상)

프로젝트: 관심 필지 지도 (Parcel Explorer)  
작성일: 2026-03-12  
권장 길이: 60-90초 (기준안 75초)

## 대상 독자
- 지자체 재산관리 부서 실무자
- 공공 데이터 운영 담당자
- 내부 보고/시연용 소개 영상을 준비하는 행정 실무자

## 목적
- 시청 10초 이내에 서비스 가치가 `공공 정보 접근성 + 실무 운영 효율`이라는 점을 전달한다.
- 공개 지도 탐색, 공개 다운로드, 모바일 활용, 관리자 최신화 흐름을 한 번씩 보여준다.
- 기능 나열보다 주민 공개 가치와 실무 연계 효과를 우선 설명한다.

## 제작 사양
- 화면비: 16:9 (기본), 필요 시 9:16 재편집
- 영상 템포: 컷당 3-6초, 기능 시연 컷 6-12초
- 자막 규칙: 한 줄 15자 내외, 명사보다 동사 중심
- 화면 원칙: 검색 결과, 목록, 상세 정보, 다운로드, 관리자 갱신 흐름이 실제 제품 순서와 일치해야 한다.
- 보안 규칙: 계정, 비밀번호, 세션/토큰, 개인정보, 내부 경로, 민감한 운영값 노출 금지

## 샷리스트 (75초)
| 구간 | 화면(조작) | 자막 | 내레이션 |
|---|---|---|---|
| 0-8초 | 메인 지도 첫 화면 진입. 지도, 검색 진입 버튼, 기본 화면 구성을 한 번에 보여주기 | 공유재산 현황, 지도에서 바로 확인 | 관심 필지 지도로 공개 가능한 자산 정보를 한 화면에서 빠르게 파악할 수 있습니다. |
| 8-22초 | 검색 오버레이 또는 필터 영역 열기. 주소, 면적, 재산 구분 등 조건 입력 후 결과 패널과 지도 강조 | 조건 검색으로 대상 필지 빠르게 찾기 | 담당자는 필요한 조건으로 관심 필지를 좁혀 보고, 지도와 목록을 함께 보며 바로 검토할 수 있습니다. |
| 22-34초 | 결과 목록 선택, 지도 피처 강조, 상세 정보 확인. 필요 시 팝업 또는 상세 패널 노출 | 목록과 지도가 함께 움직이는 검토 흐름 | 검색 결과를 선택하면 위치와 상세 정보가 함께 연결되어 현황 설명과 내부 검토가 쉬워집니다. |
| 34-45초 | 전체 목록 다운로드 또는 공개 다운로드 흐름 시연. 공개 자료 접근 장면을 간결하게 제시 | 공개 자료 제공도 일관되게 | 주민 공개나 부서 간 공유가 필요한 자료도 동일한 기준으로 제공해 정보 전달의 일관성을 높입니다. |
| 45-57초 | 모바일 화면 흐름 시연. 초기 화면에서 검색 진입, 결과 패널 확인까지 짧게 연결 | 현장에서도 같은 흐름으로 확인 | 모바일에서도 검색과 결과 확인 흐름을 유지해 현장 확인이나 간단한 설명 업무에 바로 활용할 수 있습니다. |
| 57-69초 | 관리자 로그인 후 업로드, 데이터 갱신, 통계 또는 경계선 관리 화면을 짧게 시연 | 관리자 화면으로 최신 데이터 유지 | 관리자 화면에서는 엑셀 업로드와 운영 관리 기능으로 공개 정보를 최신 상태로 유지할 수 있습니다. |
| 69-75초 | 메인 지도 복귀. 공개 지도, 다운로드, 관리자 운영 화면을 빠르게 교차 편집 후 종료 | 공개성, 최신성, 실무 효율을 하나로 | 관심 필지 지도는 공개 정보 접근성과 공공 실무의 운영 효율을 하나의 워크플로로 연결합니다. |

## 편집 메모
- 오프닝 10초 안에 `무엇을 보여주는 서비스인지`와 `누가 왜 쓰는지`가 동시에 드러나야 한다.
- 자막은 주민 공개 가치와 실무 효율 메시지를 분리해 과도한 반복을 피한다.
- 관리자 장면은 보조 흐름으로 다루고, 공개 지도와 공개 활용 장면을 전면에 둔다.
- 전환 효과는 짧게 유지하고, 기능 컷에서는 지도와 결과 패널의 동기화가 보이도록 멈춤 시간을 확보한다.
- 더미 데이터 또는 비식별 데이터로 촬영하고 실제 운영 계정 정보는 사용하지 않는다.

## 검수 체크리스트
- 10초 이내에 서비스 목적이 이해되는가?
- 공공 정보 공개 가치와 실무 운영 가치가 함께 드러나는가?
- 공개 탐색, 상세 확인, 공개 활용, 모바일 활용, 관리자 최신화 흐름이 모두 포함되는가?
- 실제 제품 기능 범위를 넘는 장면이나 문구가 없는가?
- 계정, 개인정보, 보안 설정값 등 민감정보 노출이 없는가?
- 75초 기준에서 메시지가 과밀하지 않고 이해 가능한가?
